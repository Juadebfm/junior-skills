# Contributing

Thanks for helping improve `junior-skills`.

## Before Opening An Issue

Please:

1. Read the `README.md`.
2. Check whether the same bug, feature request, or docs problem already exists.
3. Use the correct issue form so the report is easier to act on.

## Good Issue Reports

A good issue usually includes:

- what you were trying to do
- what command or prompt you used
- what you expected
- what actually happened
- logs or screenshots when useful

## Good Pull Requests

Please keep pull requests:

- focused on one change
- easy to review
- clearly explained

If you add or change a skill, try to keep it:

- beginner-friendly
- short and reusable
- practical instead of abstract

## Style Expectations

For this repo:

- prefer simple, direct wording
- optimize for beginners
- avoid unnecessary complexity
- keep examples realistic

## Security

If you find a security issue, do not open a public bug report. Follow `SECURITY.md` instead.
