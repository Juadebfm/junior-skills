Use this skill when the user wants to grow a project after the first version is clear.

## Goal

Help the user expand the project in a controlled way.

## Workflow

### Step 1

Identify what the user wants to add:

- new pages
- new features
- more reusable components
- more backend logic
- more data
- more complexity or abstraction

### Step 2

Separate the requested growth into:

- good to add now
- better to add later
- too advanced for the current stage

### Step 3

Produce:

## Scope Growth Plan

- New items requested
- What each item changes
- What should happen now
- What should wait
- Risks of adding too much too early

## Next Version

- version 1.1 items
- version 1.2 items
- version 2 items

### Step 4

If the user asks for abstraction, explain whether it is truly needed now or if simple duplication is better for the current stage.

## Rules

- Protect the beginner from overbuilding.
- Prefer phased growth over giant rewrites.
- Keep recommendations practical and ordered.
- Say clearly when a request is too much for the current project stage.
