Use this skill after a feature or page has been built.

## Goal

Give the user a simple, honest review that helps them improve the work without overwhelming them.

## Review Focus

Check for:

- broken behavior
- missing empty or error states
- layout issues on small screens
- confusing labels or copy
- form issues
- obvious accessibility issues
- code that is much more complex than needed

## Output

Present findings first.

Use this structure:

## Findings

- Issue
- Why it matters
- What to fix next

## Quick Wins

- easiest improvement
- second easiest improvement

## Good Progress

- one or two things the user did well

## Rules

- Be kind but direct.
- Prioritize real problems over style preferences.
- Keep the number of findings small and useful.
- If there are no clear issues, say that explicitly.
