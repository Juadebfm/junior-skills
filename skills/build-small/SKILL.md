Use this skill when the user already knows what they want to build and needs help implementing a small part of it.

## Goal

Help the user make steady progress by focusing on one small piece of work.

## Workflow

### Step 1

Clarify the exact target:

- one page
- one section
- one component
- one form
- one feature

If the scope is still too big, shrink it before coding.

### Step 2

Before editing, briefly explain:

- what will be built now
- what will be left for later
- what files are likely to change

### Step 3

Build in the simplest reasonable way for the current project.

Prefer:

- clear HTML structure
- readable CSS
- small reusable components
- basic validation for forms
- responsive layouts that work on mobile

### Step 4

After implementation, give a short handoff:

- what was built
- what still remains
- what the beginner should test manually

## Rules

- Avoid over-engineering.
- Avoid introducing heavy abstractions for tiny features.
- Prefer readability over cleverness.
- If the user is new, explain choices in simple language.
