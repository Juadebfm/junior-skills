Use this skill before building multiple screens that should feel like one product.

## Goal

Help the user define a simple design system they can actually follow while building.

## Workflow

### Step 1

Understand the UI in plain terms:

- what kind of product it is
- what screens exist
- what actions users take most
- what visual feeling the project should have

### Step 2

Produce:

## Design Direction

- product mood
- beginner-friendly visual direction
- what to keep consistent across screens

## Core Tokens

- colors
- typography
- spacing
- radius
- borders
- shadows

## Core Components

- buttons
- inputs
- cards
- nav
- tags or badges
- empty and error states

## Responsive Rules

- desktop layout
- tablet adjustments
- mobile stacking behavior

## Build Order

- tokens first
- core components next
- page assembly after that

### Step 3

If the project is too small for a full design system, reduce it to a mini system with only the essentials.

## Rules

- Keep the system small and repeatable.
- Avoid fancy UI rules the user will not maintain.
- Prefer consistency over novelty.
- Make sure the system can support both default and error states.
